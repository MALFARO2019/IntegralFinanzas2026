# Manual de referencia (resumen) – YNAB
Fecha: 2026-02-25

> Guía resumida basada en páginas oficiales de YNAB (Help Center + Guides).

## 1) Conceptos clave
- **Plan (presupuesto por categorías)**: el usuario planifica asignación por categorías (enfoque YNAB).
- **Targets**: metas por categoría (monto/cadencia/due date) para planificar gastos futuros.
- **Scheduled Transactions**: transacciones futuras/repetitivas; se integran al flujo de planificación.
- **Auto-Assign**: asignación opcional que usa Targets y Scheduled Transactions para distribuir fondos.
- **Linked Accounts (Direct Import)**: importar transacciones conectando banco (opcional).
- **File-Based Import**: importar sin conectar banco (archivo desde el banco; web/iPad según YNAB).

## 2) Primeros pasos (setup recomendado)
1. Definir categorías del plan.
2. Configurar **Targets** para categorías importantes (gastos fijos, gastos no mensuales).
3. Crear **Scheduled Transactions** para bills recurrentes.
4. Elegir método de ingreso de transacciones:
   - Manual
   - Linked accounts (Direct Import)
   - File-based import (archivo)

## 3) Transacciones
### 3.1 Agregar y organizar
- Registro de transacciones (manual o import).
- **Split transactions**: dividir una transacción entre múltiples categorías/payees (guía oficial).
- **Flags**: marcar y agrupar transacciones (reembolsos, pendientes, “revisar”, etc.) con codificación por color (guía oficial).

### 3.2 Importación
- Linked accounts usan Direct Import.
- File-based import permite importar desde archivo sin conexión bancaria.
- Hay guías oficiales para formatear CSV y para file-based import.

## 4) Asignación (Auto-Assign)
- Auto-Assign utiliza la configuración de Targets y Scheduled Transactions para priorizar asignación (guía oficial).

## 5) Reconciliación
- YNAB ofrece guías oficiales de reconciliación y troubleshooting.
- Existe una mejora descrita por YNAB: “Reconcile with Direct Import” para comparar balance importado con el del banco y resaltar discrepancias.
